# Retro Outcomes

We used a simple discussion-based retrospective where each team member shared reflections on what worked well during the quarter, areas where the team could improve, and priorities for the final stretch before deployment and presentation.

Report:

* successes:
  * The team maintained consistent communication through standups, which helped keep everyone aligned on progress.
  * Our task breakdown and GitHub issue tracking helped distribute work clearly across team members.
  * Collaboration across components (scraping, embeddings, backend, and frontend) improved as the project progressed.
  * Documentation such as the README, DESIGN, and testing files helped organize the project and made onboarding easier.
  * The team successfully built a working end-to-end pipeline for the GauchoGuider system, including data ingestion, vector search, and response generation.

* a goal:
  * Improve coordination when integrating different components of the system so that testing and debugging can happen earlier and more smoothly.

* a commitment:
  * The team commits to prioritizing final integration, testing, and documentation early so that the final demo and deployment are stable and clearly explained.

# Optional

As we move toward launch, the team agreed to focus on polishing the user experience, ensuring the documentation clearly explains the system architecture, and preparing a clear presentation of how GauchoGuider works and what value it provides to UCSB students.
